# CLAUDE.md — anthemic-hub

## What this repo is

Static site for `anthemic-developments.com`. No build step, no framework — plain HTML/CSS/JS plus a PHP admin panel running in Docker on the Droplet.

## Key facts

- **No build step.** All HTML/CSS/JS is hand-written. Never introduce a bundler or framework.
- **Two repos:** this repo (`anthemic-hub`) owns site content; `anthemic-ops` (sibling dir) owns nginx config. Changes to nginx go in anthemic-ops.
- **Deploy:** push to `main` → GitHub Actions → rsync to Droplet → apply script promotes files. See `.github/workflows/deploy.yml` and `scripts/droplet/anthemic-hub-deploy-apply.sh`.
- **Three files are preserved across deploys** (git copy is seed only): `gigs/gigs.json`, `content/hub.json`, `assets/gallery/manifest.json`.

## Admin panel

- URL: `/gigs/admin/`
- Auth: nginx `auth_basic` (htpasswd on Droplet) + PHP bcrypt password (Docker env var)
- Runs as: `php:8.2-cli` container, port `127.0.0.1:9001`, volume `/var/www/anthemic-hub:/app`
- Entry: `/app/gigs/admin/index.php` with `-c /app/gigs/admin/php.ini`
- Tabs: Gig calendar · Site content · Gallery
- CSRF token in session on all POST actions

## Dynamic content (fetched client-side)

| File | What it drives |
|------|---------------|
| `/gigs/gigs.json` | Gig calendar page (`gigs/index.html`) |
| `/content/hub.json` | Hub bio text, instruments, projects list |
| `/assets/gallery/manifest.json` | Photo gallery on hub landing page |

All fetches have HTML fallback content — page works without JS.

## File paths inside the PHP container

The Docker volume mounts `/var/www/anthemic-hub` as `/app`. PHP file is at `/app/gigs/admin/index.php` so `__DIR__` = `/app/gigs/admin`.

| PHP `$var` | Resolves to |
|------------|-------------|
| `$gigsPath` | `/app/gigs/gigs.json` |
| `$contentPath` | `/app/content/hub.json` |
| `$galleryPath` | `/app/assets/gallery` |

## hub.json schema

```json
{
  "who_am_i": "string",
  "music_bio_origin": ["paragraph", "paragraph"],
  "music_bio_anthems": ["paragraph", "paragraph"],
  "instruments": [{ "name": "string", "primary": true }],
  "projects": [{ "name": "string", "dates": "string" }]
}
```

## Gallery manifest schema

Auto-generated by the admin on every upload/delete. Do not edit manually.

```json
{ "images": ["filename.jpg", "filename2.png"] }
```

## Interest filter

Cards have `data-interests="music gigs teaching creative work community personal"`. Hub sections have `data-interests` too. JS in `index.html` sorts matching content to the top when the dropdown changes. `sectionMatchCount()` checks both section-level and card-level attributes.

## nginx (anthemic-ops)

`/gigs/admin/` proxies to `127.0.0.1:9001` with `auth_basic`. Do not add PHP processing to the main nginx config — all PHP goes through the container proxy.

## What NOT to do

- Do not add a JS framework or build step
- Do not add comments explaining what code does — only add comments for non-obvious WHY
- Do not modify `swarm-reporter` nginx config — it is a separate site on the same Droplet
- Do not commit htpasswd files or password hashes
- Do not edit `assets/gallery/manifest.json` manually — it is auto-generated
